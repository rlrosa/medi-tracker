# Native Android App Support

We have successfully configured the project to support an Android application utilizing Capacitor.
Capacitor acts as a wrapper that allows us to build a native application that essentially embeds our Next.js web application.

Here is a guide on how to work with the Android project and how to get it onto the Play Store.

## Setup Instructions

1. **Install Android Studio**
   Download and install [Android Studio](https://developer.android.com/studio). This is required to compile the Android app and submit it to the Google Play Store.

2. **Setup Android SDKs**
   Open Android Studio and ensure you have installed the required SDKs (API level 34 or higher is recommended).

3. **Install Capacitor Dependencies**
   If you clone this repository fresh, make sure you run:
   ```bash
   npm install
   ```

4. **Web Build**
   The Capacitor app wraps a build of your Next.js frontend. Although we are linking to a live server for the majority of the app, we need an initial build folder for Capacitor.
   ```bash
   mkdir out
   echo '<!DOCTYPE html><html><head><title>MediTracker</title></head><body></body></html>' > out/index.html
   ```

5. **Sync Capacitor**
   Sync your web code and capacitor configuration into the Android project.
   ```bash
   npx cap sync android
   ```

6. **Open in Android Studio**
   You can easily open the Android project in Android Studio by running:
   ```bash
   npx cap open android
   ```
   Alternatively, open Android Studio and choose "Open an existing project" and select the `android` folder in the root of this repository.

## Preparing for the Google Play Store

1. **Update App Credentials**
   Open `capacitor.config.ts`.
   - Update `appId` to your final unique application ID (e.g., `com.yourcompany.meditracker`).
   - The `appName` is the name displayed to users.
   - Run `npx cap sync android` after changing these values.
   - **Important**: If you change the `appId`, you may need to refactor the Android package names in Android Studio (`MainActivity.java` and `build.gradle`).

2. **Update App Icons and Splash Screens**
   Use `cordova-res` or `@capacitor/assets` to automatically generate icons and splash screens.
   ```bash
   npm install -g @capacitor/assets
   npx capacitor-assets generate --android
   ```
   You'll need to provide an icon (`resources/icon.png`) and splash screen (`resources/splash.png`).

3. **Configure Permissions**
   If you need specific native permissions (like camera, push notifications, etc.), you'll need to add them to `android/app/src/main/AndroidManifest.xml` and potentially install specific Capacitor plugins (e.g., `@capacitor/push-notifications`).

4. **Sign the App**
   To submit to the Play Store, you must sign your APK or App Bundle (AAB).
   - In Android Studio, go to `Build > Generate Signed Bundle / APK`.
   - Choose `Android App Bundle` (this is required by Google Play now).
   - Create a new keystore or use an existing one. Keep the keystore file secure! If you lose it, you can't update your app.
   - Enter your keystore password and alias details.
   - Select the `release` build variant and click Finish.

5. **Create a Google Play Console Account**
   - Go to the [Google Play Console](https://play.google.com/apps/publish) and register for a developer account (requires a one-time $25 fee).

6. **Create an App in Play Console**
   - Create a new app and fill in the store listing details (title, short description, full description, screenshots, privacy policy).

7. **Upload the AAB**
   - Go to `Testing > Internal testing` or `Production` and create a new release.
   - Upload the `.aab` file generated by Android Studio (usually located in `android/app/release/app-release.aab`).

8. **Review and Publish**
   - Fill out the content rating questionnaire.
   - Complete the app content declarations (e.g., Target Audience, Data Safety).
   - Roll out the release to review!
